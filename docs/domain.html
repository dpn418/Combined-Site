Domain Names and IP Addresses

Domain names?

Domain names are what you type in your web browser's address bar when you want to go to a website. These alphanumeric names are connect to the numerical IP addresses of the machines that you are accessing web pages from. A domain name may have multiple IP addresses associated with it, but IP addresses will only have one domain tied to it. For example google.com uses many IP addresses to handle the load of traffic to such a popular website. Domain names are important because it is much more convenient to remember a name with letters, like "google.com", than it is to remember a numerical IP address, such as 64.233.191.255 .

DNS?

DNS stands for Domain Name System. It is what translates easily-read domain names like google.com to the numerical IP addresses that machines use. It is like a phonebook where instead of people's names and phone numbers, there are domain names and IP addresses. Your computer stores these DNS records and updates them by sending a request to your local DNS server whenever your current records are out of date or your computer does not have the record that it needs.

IP Addresses?

The IP in IP address stands for "Internet Protocol". An IP address is a unique set of 4 numbers, separated by periods, used to indentify where a device is on the Internet. IP addresses are automatically assigned by your Internet Service provider. The protocol for assigning IP addresses and the organization that allots a range of IP addresses to your ISP is ICANN. IP addresses work like physical addresses in the way that they provide a way for information to make it to a device through the Internet the way mail makes it to a mailbox.

Who is ICANN?

ICANN stands for the Internet Corporation for Assigned Names and Numbers. They are responsible for the coordination of how unique IP addresses and domain names are assigned. They are also are the central repository for IP addresses that get assigned in ranges to different regions. For more about ICANN, visit their website.

Packets and Routing?

Packets are small segments of information, usually part of a larger message, that get sent over the internet. Packets are necessary to help with scheduling the use of paths along the internet and for finding the best path between two points. Routing is the way that paths are selected for data packets to travel along. There are usually may ways for data to get from one device to another via the Internet, but some wires or devices may be in use or take longer than others.

What are HTTP and HTTPS?

HTTP stands for HyperText Transfer Protocol and it is the application layer of the Internet that allows for webpages to be loaded on your computer with a hyperlink. HTTP involves sending requests for information to be sent and waiting for those requests in the header of data packets. HTTPS is the secure version of HTTP and secure is what the S in HTTPS stands for. HTTPS is encrypted, meaning the data has to be unraveled with a key before being read, while HTTP is not.
